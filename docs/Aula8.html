INTRODUÇÃO

ONDE ESTAMOS

  • Na última aula, aprendemos a modelar tempos de sobrevivência/confiabilidade com modelos paramétricos:

    • Exponencial, Weibull, Log‑normal, Gama e Gama generalizado.
    • Vimos a estimação pontual e intervalar dos parâmetros desses modelos e funções associadas.

  • Todos esses modelos são candidatos possíveis para um mesmo conjunto de dados.

  • Toda a inferência a partir do método de máxima verossimilhança para as quantidades de interesse só poderá ser devidamente interpretada após ter sido definido um modelo probabilístico adequado para os dados.

  • Mas surge uma pergunta natural: Qual modelo escolher?

QUAL MODELO ESCOLHER?

  • Cada modelo tem uma forma diferente de representar:

    • O comportamento do tempo de ocorrência do evento
    • A evolução da taxa de risco ao longo do tempo

  • Com os dados em mãos, precisamos decidir qual deles se ajusta melhor

  • Essa escolha não é óbvia — diferentes modelos podem dar respostas diferentes

POR QUE NÃO DÁ PARA SABER ANTES?

  • Não sabemos, a priori, qual é a verdadeira forma da distribuição dos tempos

  • Por exemplo, somente após definir que o modelo log-normal se ajusta bem aos dados é que podemos fazer as devidas interpretações das quantidades de interesse estimadas pelo método de máxima verossimilhança

  • Entretanto, se o modelo log-normal for usado inadequadamente para um certo conjunto de dados:

    • Toda a análise estatística fica comprometida
    • As respostas às perguntas de interesse ficam distorcidas

  • Chegamos a um problema clássico de seleção de modelos em estatística

COMO VAMOS PROCEDER?

  • Nessa aula, veremos seis métodos que, conjuntamente, contribuem para comparar e escolher o melhor modelo paramétrico:

    1. Método gráfico baseado na curva de Kaplan‑Meier.
    2. Método gráfico da linearização da função de sobrevivência.
    3. Critérios de informação: AIC e BIC.
    4. Teste da razão de verossimilhança (para modelos aninhados).
    5. Teste de Kolmogorov Smirnov para dados censurados.
    6. Conversa com especialista.

  • Ao final, você terá um conjunto de ferramentas para fazer uma escolha fundamentada

MÉTODO GRÁFICO: KAPLAN-MEIER

INTRODUÇÃO

  • O método gráfico mais utilizado consiste na comparação da função de sobrevivência (confiabilidade) do modelo proposto com o estimador de Kaplan-Meier

  • Neste procedimento, ajustam-se os modelos paramétricos propostos ao conjunto de dados e, a partir das estimativas dos parâmetros de cada modelo, estimam-se suas respectivas funções de sobrevivência

  • Para o conjunto de dados, obtém-se também a estimativa de Kaplan-Meier para a função de sobrevivência

PROCEDIMENTO

  • Finalmente, comparam-se graficamente as funções de sobrevivência estimadas para cada modelo proposto com a função de sobrevivência estimada pelo método de Kaplan-Meier

  • O(s) modelo(s) adequado(s) é(são) aquele(s) em que sua curva de sobrevivência mais se aproximar daquela do estimador de Kaplan-Meier

  • Em outras palavras, o(s) melhor(es) modelo(s) será(ão) aquele(s) cujos pontos no gráfico estiverem mais próximos da reta y = x

GRÁFICO ALTERNATIVO

  • Uma outra forma de comparar as funções é colocar, em um mesmo gráfico, todas as curvas de sobrevivência estimadas versus o tempo t

  • Alguns autores, como Nelson (1990), sugerem a comparação da função de risco acumulada estimada de forma não paramétrica com a função de risco acumulada estimada pelos modelos probabilísticos

    • Entretanto, na prática, essa abordagem é menos comum.

VANTAGENS E DESVANTAGENS

  • Como um método gráfico, ele representa apenas um primeiro filtro para o ajuste dos modelos probabilísticos

  • Ele é capaz de detectar facilmente os modelos que não se ajustam bem aos dados

  • Caso nenhum modelo se adeque bem aos dados nessa abordagem, pode ser sugerida uma análise estatística toda baseada em técnicas não paramétricas

  • A grande desvantagem dessa abordagem é que as conclusões dela podem diferir para diferentes analistas

    • Isto é, existe uma componente subjetiva na sua interpretação

MÉTODO GRÁFICO: LINEARIZAÇÃO

INTRODUÇÃO

  • Este método consiste na linearização da função de sobrevivência obtida por Kaplan-Meier

  • Ideia básica: construir gráficos que sejam aproximadamente lineares caso o modelo proposto seja apropriado

  • Violações da linearidade podem ser rapidamente verificadas visualmente

  • O gráfico utilizado é de uma transformação que lineariza a função de sobrevivência do modelo proposto

  • Resultado final: uma reta se o modelo proposto for adequado

ATENÇÃO: NÃO FUNCIONA PARA TODOS

  • Este método funciona para modelos específicos: Exponencial, Weibull e Log-normal

  • Distribuições como Gama e Gama Generalizada não possuem linearização simples

  • Para esses casos, outros critérios (AIC, BIC, TRV) serão necessários

  • O método da linearização é um diagnóstico visual, não um teste formal

LINEARIZAÇÃO - MODELO EXPONENCIAL

  • Para o modelo exponencial, a função de sobrevivência é:

\[S(t) = e^{-\alpha t}\]

Assim:

\[-\log[S(t)] = \alpha t\]

  • Consequentemente, \(-\log[\hat{S}(t)]\) é uma função linear de \(t\) (passando pela origem)

  • Gráfico a ser feito: \(-\log[\hat{S}(t)]\) versus \(t\) (\(\hat{S}(t)\) é o estimador de Kaplan-Meier).

  • Deve ser aproximadamente linear se o modelo exponencial for apropriado

LINEARIZAÇÃO - MODELO WEIBULL

  • Para o modelo Weibull com parâmetros \((\kappa, \gamma)\):

\[S(t) = e^{-\left(\frac{t}{\kappa}\right)^\gamma}\]

Transformação:

\[\log\{-\log[S(t)]\} = -\gamma\log(\kappa) + \gamma\log(t)\]

  • \(\log\{-\log[\hat{S}(t)]\}\) é função linear de \(\log(t)\)

  • Gráfico a ser feito: \(\log\{-\log[\hat{S}(t)]\}\) versus \(\log(t)\) (\(\hat{S}\) é o estimador de Kaplan-Meier).

  • Deve ser aproximadamente linear se o modelo Weibull for apropriado

LINEARIZAÇÃO - MODELO LOG-NORMAL

Para o modelo log-normal com parâmetros \((\mu, \sigma)\):

\[S(t) = \Phi\left(-\frac{\log(t) + \mu}{\sigma}\right)\]

Transformação:

\[\Phi^{-1}[S(t)] = -\frac{\log(t) + \mu}{\sigma} = -\frac{\mu}{\sigma} - \frac{1}{\sigma}\log(t)\]

  • \(\Phi^{-1}[\hat{S}(t)]\) é função linear de \(\log(t)\) (\(\hat{S}\) é o estimador de Kaplan-Meier).

  • \(\Phi^{-1}(\cdot)\) são os percentis da distribuição Normal padrão

  • Gráfico a ser feito: \(\Phi^{-1}[\hat{S}(t)]\) versus \(\log(t)\)

  • Ele é capaz de detectar facilmente os modelos que não se ajustam bem aos dados

CRITÉRIOS DE INFORMAÇÃO AIC E BIC

CRITÉRIO DE INFORMAÇÃO DE AKAIKE (AIC)

  • AIC (Critério de Informação Akaike) é geralmente considerado como um dos primeiros critérios de seleção de modelo

  • Hoje, o AIC continua a ser a mais amplamente conhecida e usada ferramenta de seleção de modelos entre os profissionais

  • Foi introduzido por Hirotugu Akaike em 1973

  • É uma métrica que mensura a qualidade de um modelo estatístico visando também a sua simplicidade (parcimônia)

  • O AIC leva em consideração e penaliza a complexidade dos modelos, favorecendo modelos mais simples

CRITÉRIO DE INFORMAÇÃO DE AKAIKE (AIC)

  • Considere:
    • Um determinado modelo estatístico ajustado aos dados
    • \(k\) = número de parâmetros do modelo
    • \(\hat{L}\) = valor máximo da função de verossimilhança
  • O valor do AIC é dado por:

\[AIC = 2k - 2\ln(\hat{L})\]

  • Regra de escolha: Dada uma coleção de modelos candidatos, o modelo com menor AIC é o escolhido de acordo com este critério

  • O AIC bonifica a qualidade de ajuste (altos valores da verossimilhança) e penaliza a quantidade de parâmetros do modelo

  • Fornece uma métrica para comparação e seleção de modelos: menores valores de AIC representam maior qualidade e simplicidade

CRITÉRIO DE INFORMAÇÃO BAYESIANO (BIC)

  • Conhecido como Critério de Informação Bayesiano (BIC)

  • É uma métrica estatística para selecionar o melhor modelo em uma análise de dados

  • Baseado na teoria da probabilidade bayesiana

  • Amplamente utilizado em áreas como machine learning, deep learning e inteligência artificial

  • Busca equilibrar a complexidade do modelo e a qualidade do ajuste, penalizando modelos mais complexos para evitar overfitting

CRITÉRIO DE INFORMAÇÃO BAYESIANO (BIC)

  • Considere:
    • Modelo estatístico ajustado a \(n\) dados observados
    • \(k\) = número de parâmetros do modelo
    • \(\hat{L}\) = valor máximo da função de verossimilhança
  • O valor do BIC é dado por:

\[BIC = k \cdot \log(n) - 2\ln(\hat{L})\]

  • Regra de escolha: Dada uma coleção de modelos candidatos, o modelo com menor BIC é o escolhido de acordo com este critério

COMPARAÇÃO ENTRE AIC E BIC

  • Diferença fundamental:

    • AIC penaliza com \(2k\) (constante)
    • BIC penaliza com \(k \cdot \log(n)\) (cresce com \(n\))

  • Consequência:

    • Para \(n > 7\), \(\log(n) > 2\) → BIC penaliza mais que o AIC
    • Quanto maior \(n\), maior a penalidade do BIC em relação ao AIC

  • Tanto o AIC como o BIC não fornecem uma medida de qualidade do modelo global, apenas relativa à comparação entre modelos candidatos

  • Se todos os modelos propostos se ajustam mal aos dados, o AIC e BIC não explicitam tal fato

TESTE RAZÃO DE VEROSSIMILHANÇA

INTRODUÇÃO

  • O Teste da Razão de Verossimilhança (TRV) é um método formal para comparar modelos aninhados (encaixados)

  • Um modelo é aninhado dentro de outro quando é um caso particular deste

  • Estrutura de modelos aninhados que vimos:

    • (Exponencial, Weibull, Gama e Lognormal) ⊂ Gama Generalizado

  • O modelo Gama Generalizado é o mais flexível (3 parâmetros) e generaliza os demais

  • Implicação prática:

    • O Gama Generalizado pode ser usado como modelo de partida
    • Testamos hipóteses sobre os parâmetros para decidir se um modelo mais simples é suficiente

TESTE TRV - FORMULAÇÃO GERAL

  • Considere:
    • Modelo complexo (Gama Generalizado): \(p\) parâmetros
    • Modelo simples (aninhado): \(q\) parâmetros, com \(q < p\)
  • Hipóteses:
    • \(H_0\): o modelo simples é mais adequado.
    • \(H_1\): o modelo complexo é mais adequado.

Estatística do teste: \[\Lambda = 2[\ln L(\text{complexo}) - \ln L(\text{simples})]\]

Distribuição sob \(H_0\): \[\Lambda \sim \chi^2_{df} \quad \text{com} \quad df = p - q\]

TESTE TRV - REGRA DE DECISÃO

  • Procedimento:
    1. Ajustar o modelo complexo (Gama Generalizado) → obtenha \(\log L_{\text{complexo}}\)
    2. Ajustar o modelo simples (Exponencial, Weibull, Gama ou Log-normal) → obtenha \(\log L_{\text{simples}}\)
    3. Calcular: \(\Lambda = 2(\log L_{\text{complexo}} - \log L_{\text{simples}})\)
    4. Comparar com o quantil da \(\chi^2\) com \(df = p - q\)
  • Regra de decisão:
    • Se \(\Lambda > \chi^2_{df, 1-\alpha}\) → rejeita \(H_0\) → prefere o modelo complexo
    • Se \(\Lambda \leq \chi^2_{df, 1-\alpha}\) → não rejeita \(H_0\) → modelo simples é suficiente

TESTE TRV - CUIDADOS E LIMITAÇÕES

  • Atenção:
    • O TRV só é válido para modelos aninhados (um é caso particular do outro)
    • Não pode ser usado para comparar, por exemplo, Weibull vs Log-normal (não são aninhados)

TESTE DE KOLMOGOROV SMIRNOV

TESTE DE KOLMOGOROV-SMIRNOV PARA DADOS CENSURADOS

  • O teste de Kolmogorov-Smirnov (KS) é um clássico teste de aderência (goodness-of-fit) que compara:

    • A distribuição empírica observada
    • Versus uma distribuição teórica especificada

  • Problema: A versão tradicional do teste KS não funciona na presença de censura

  • Solução: Foram propostas adaptações para dados censurados à direita, como a versão de Fleming, O’Fallon e O’Brien (1980)

TESTE KS PARA DADOS CENSURADOS

  • Implementação no R: pacote GofCens com a função KScens()

  • Distribuições suportadas: Exponencial, Weibull, Log-normal, Log-logística, Gumbel, Normal, Logística e Beta

Por que não ensinaremos este teste neste curso?

  • Depende de conhecimentos ainda não vistos: bootstrap e inferência não paramétrica avançada

  • O cálculo dos p-valores geralmente requer reamostragem (bootstrap)

  • Por razões didáticas, focaremos nos métodos anteriores (gráficos, AIC/BIC e TRV)

  • Fica como tópico complementar para estudos futuros

CONVERSA COM ESPECIALISTA

A CONVERSA COM O ESPECIALISTA

  • Nem toda escolha de modelo vem apenas de critérios estatísticos

  • O conhecimento prático de quem conhece o fenômeno estudado é fundamental

  • O especialista pode trazer informações que os dados, sozinhos, não revelam

  • Combinar estatística + conhecimento de domínio → decisões mais robustas

O QUE O ESPECIALISTA PODE DIZER?

  • Sobre o processo de ocorrência do evento:
    • A taxa de risco é constante ao longo do tempo? (Exponencial)
    • A taxa de risco aumenta com o tempo? (Weibull com \(\gamma > 1\))
    • A taxa de risco diminui com o tempo? (Weibull com \(\gamma < 1\))
    • O risco é não monotônico (cresce e depois cai)? (Log-normal ou Gama Generalizado)
  • Sobre o fenômeno:
    • Existe um período de amadurecimento inicial?
    • Componentes envelhecem ou melhoram com o tempo?

COMO EXTRAIR INFORMAÇÕES?

Mesmo sem acesso direto a um especialista, vocês podem:

  • Ler a literatura técnica da área de aplicação (engenharia, medicina, biologia, etc.)

  • Buscar artigos científicos que estudaram fenômenos semelhantes

  • Conversar com profissionais da área (supervisores de estágio, professores de outras disciplinas, profissionais do setor)

  • Observar o comportamento empírico dos dados (via Kaplan Meier, gráficos de risco, etc.)

  • Consultar relatórios técnicos e manuais de equipamentos
